# Variant Strategy

## Good variant axes

- Size
- Visual style
- State
- Density when it changes layout meaningfully

## Avoid bad axes

- Raw color choice when the semantic intent is already captured elsewhere
- Text copy differences
- Icon choice
- Content length

## Split rules

Split into separate components when:

- Variant combinations exceed what a designer can scan quickly.
- Different variants need different anatomy.
- A variant only exists for one narrow use case.
- One axis is really a nested sub-component concern.
