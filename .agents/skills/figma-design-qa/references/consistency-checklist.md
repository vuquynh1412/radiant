# Consistency Checklist

## Spacing

- Gaps align to the spacing scale.
- Related sections use the same padding logic.
- Dense and spacious layouts are applied intentionally, not randomly.

## Typography

- Heading hierarchy is consistent.
- Body and helper text use the intended styles.
- Line length and wrapping remain readable across screen sizes.

## Components and states

- The same component behaves the same way in comparable contexts.
- States are not missing in one screen and present in another.
- Icons, badges, and support text use consistent placement patterns.

## Responsive behavior

- Layout changes follow the documented breakpoint logic.
- Controls wrap or stack consistently.
- No screen relies on desktop-only adjacency for basic comprehension.

## Token usage

- Colors and effects map back to system tokens.
- One-off values are documented and intentional.
