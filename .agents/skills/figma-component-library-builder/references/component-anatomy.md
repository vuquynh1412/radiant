# Component Anatomy

## Define structure first

- Root container
- Primary content
- Optional supporting content
- Optional iconography
- Status or feedback region
- Interaction affordances

## Good anatomy characteristics

- Slots are obvious.
- Content and decoration are separated.
- States can be expressed without duplicating the whole component.
- The component can resize without breaking alignment.

## Common web components

- Button: container, label, optional leading icon, optional trailing icon, optional loader
- Input: field shell, input text, label, helper text, error text, optional prefix or suffix
- Card: container, heading group, body content, actions, optional media or status region
