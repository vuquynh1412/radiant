# Screen Composition

## Build order

1. Page wrapper
2. Global navigation or local header
3. Primary hero or page intro
4. Main content sections in priority order
5. Supplemental panels
6. Footer or persistent actions

## Section rules

- Give each section one job.
- Keep content hierarchy obvious inside each section.
- Use component instances for repeated patterns.
- Document when a section intentionally diverges from the default token or layout system.

## State coverage

- Include empty and loading states for data-driven areas.
- Include error and validation states for forms.
- Include selected, active, or expanded states for navigation and disclosure patterns.
