# WCAG UI Checklist

## Contrast

- Text meets WCAG AA contrast for its size and weight.
- Essential icons, strokes, and control boundaries remain visible.
- Focus indicators are visible and distinct.

## Targets and controls

- Interactive targets are large enough for reliable activation.
- Icon-only controls have obvious labels or supporting context.
- Dense rows do not create accidental taps.

## States

- Focus is visible.
- Hover is not the only cue for affordance.
- Disabled, error, selected, expanded, and loading states are distinguishable.

## Forms

- Labels are persistent and unambiguous.
- Helper and error text are readable and proximate to the field.
- Required or invalid states are not communicated by color alone.

## Responsive and reflow

- No essential information depends on side-by-side layout at narrow widths.
- Keyboard focus order still makes sense when the layout stacks.
- Primary actions remain available without awkward horizontal scrolling.
