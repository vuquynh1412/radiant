# Dev Mode Handoff

## What to specify

- Which components are reused versus custom
- Which tokens or variables drive color, spacing, radius, typography, and effects
- Which sizes are fixed and why
- Which containers should wrap, fill, or stack
- Which states must be implemented
- Which breakpoint changes are expected

## Good handoff qualities

- The developer can recreate layout behavior without guessing.
- Token usage is explicit.
- Exceptions are called out instead of being implied by screenshots.
- Behavior notes are short and testable.

## Do not rely on

- Visual screenshots alone
- Ambiguous terms like "responsive" without describing the actual change
- Color names without token or semantic meaning
